üìä 1. Arrangement (Main Axis)
    ‚Ä¢ What it does: Controls how multiple children are distributed along the main axis of a layout
                    container like Row or Column.
    ‚Ä¢ Main Axis Direction:
            ‚Ä¢ Row ‚Üí Horizontal (left-to-right)
            ‚Ä¢ Column ‚Üí Vertical (top-to-bottom)
    ‚Ä¢ Used with:
            ‚Ä¢ Row: horizontalArrangement
            ‚Ä¢ Column: verticalArrangement
    ‚Ä¢ Important Constraint:
            ‚Ä¢ Row ‚ùå cannot use verticalArrangement
            ‚Ä¢ Column ‚ùå cannot use horizontalArrangement

    ‚Ä¢ üìç Types of Arrangements
        Name	                    Description
        Arrangement.Start / Top	    Children are aligned to the start (left/top).
        Arrangement.End / Bottom	Children aligned to the end (right/bottom).
        Arrangement.Center	        Children are centered within the parent.
        Arrangement.SpaceBetween	First and last children stick to the edges, others spaced evenly.
        Arrangement.SpaceAround	    Equal space around all children (half-size at start/end).
        Arrangement.SpaceEvenly	    Equal space between all children including start and end.
        Arrangement.spacedBy(dp)	Fixed spacing between children only.

üß± 2. Alignment (Cross Axis)
    ‚Ä¢ What it does: Controls how a child is positioned within its parent‚Äôs bounds, especially when:
            ‚Ä¢ There is only one child
            ‚Ä¢ You want to override default alignment behavior
    ‚Ä¢ Cross Axis Direction:
            ‚Ä¢ Row ‚Üí Vertical (top-to-bottom)
            ‚Ä¢ Column ‚Üí Horizontal (left-to-right)
    ‚Ä¢ Used with:
            ‚Ä¢ Row: verticalAlignment
            ‚Ä¢ Column: horizontalAlignment
    ‚Ä¢ Important Constraint:
            ‚Ä¢ Row ‚ùå cannot use horizontalAlignment
            ‚Ä¢ Column ‚ùå cannot use verticalAlignment

    ‚Ä¢ contentAlignment
        ‚úÖ Used in: Box
                ‚Ä¢ Aligns the content inside the Box (children as a whole)
                ‚Ä¢ Aligns both horizontally and vertically
                ‚Ä¢ Affects all children unless overridden with Modifier.align()

    ‚Ä¢ üîÑ Can be overridden per-child using Modifier.align()

    ‚Ä¢ üìç Types of Alignment
        üîπ 1. Alignment in Box ‚Üí contentAlignment & Modifier.align()
                These allow 2D alignment (horizontal + vertical). Available in:
                    ‚Ä¢ Box(contentAlignment = ...)
                    ‚Ä¢ Modifier.align(...) (used on individual children)
                ‚úÖ Common Values from Alignment:
                    Alignment Option	        Description

                    Alignment.TopStart	        Top-left corner
                    Alignment.TopCenter	        Top-center
                    Alignment.TopEnd	        Top-right corner
                    Alignment.CenterStart	    Center-left
                    Alignment.Center	        Center (middle)
                    Alignment.CenterEnd	        Center-right
                    Alignment.BottomStart	    Bottom-left
                    Alignment.BottomCenter	    Bottom-center
                    Alignment.BottomEnd	        Bottom-right

        üîπ 2. Alignment in Row ‚Üí verticalAlignment
                Used to align children vertically (cross axis of Row).

                ‚úÖ Values from Alignment.Vertical:
                    Alignment Option	            Description

                    Alignment.Top	                Align to the top
                    Alignment.CenterVertically	    Align to vertical center
                    Alignment.Bottom	            Align to the bottom

        üîπ 3. Alignment in Column ‚Üí horizontalAlignment
                Used to align children horizontally (cross axis of Column).

                ‚úÖ Values from Alignment.Horizontal:
                    Alignment Option	            Description

                    Alignment.Start	                Align to the start (LTR: left)
                    Alignment.CenterHorizontally	Align to horizontal center
                    Alignment.End	                Align to the end (LTR: right)

       üîπ 4. Text Alignment (Not layout alignment, but common confusion)
                Used in Text composables:
                Example:
                    Text(
                        text = "Hello",
                        textAlign = TextAlign.Center
                    )

                ‚úÖ Common TextAlign options:
                        TextAlign.Start
                        TextAlign.Center
                        TextAlign.End
                        TextAlign.Justify

üîÅ Rule of Thumb
    ‚Ä¢ Use Arrangement to control main axis spacing.
    ‚Ä¢ Use Alignment to control cross axis alignment.
